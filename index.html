Основные команды и заметки

1 Создать новый репозиторий на GitHub

2 Инициализация [git init]

Первой командой необходимо инициалзировать проект: кодом git init, либо нажатием кнопки в VS Code раздел Source control

3 Добавить файлы в отслеживание [git add .]

Сохраняет внесенные изменения. Точка указывает на все доступные файлы проекта

4 Коммитим проект с сообщением [git commit -m "message"]

5 При необходимости меняем название ветки на 'main' [git branch -M main]

Раньше основную ветку называли 'master'

6 Указываем ссылку на репозиторий [git remote add origin + ссылка]

Связываем локальную версию с репозиторием облака GitHub. Пример ссылки: https://github.com/seniorLion/git-lessons.git

'origin' представляет собой "коробку" с указанной ссылкой

7 Загружаем файлы проекта в репозиторий GitHub [git push -u origin main]

Файлы отправляются по ссылке 'origin' в основную ветку 'main'

8 При работе с ранее созданным проектом коммитим используя пункты №3, 4 и 7

При использовании 'UI' кнопопк VS Code

1 Инициализируем -> 2 Коммитим указывая сообщение (происходят 3 и 4 пункты выше) -> 3 Publish Branch и выбираем вверху публичный или приватный репозиторий. При этом нахвание возьмется из имени проекта

Клонирование репозитория [git clone]

git clone https://github.com/seniorLion/git-lessons.git (команда git glone + ссылка на репозиторий из гитхаба). Клонируется в то место, где открыт терминал

Состояние локального репозитория [git status]

Укажет на наличие не сохраненных/не коммиченных файлов или наоборот

Получить внесенные изменения с GitHub к себе в локальную версию [git pull]

Git Graph - просмотр истории веток

Вывод информации о коммитах в териминал: [git log --oneline --graph --decorate --all]

Создание сокращенных команд [alias]

Заходим на диск С -> Пользователи -> Файл .gitconfig

Открываем в VSC и допизсываем сокращения после команды [alias]

Пример: s = status --short

Откатиться назад к предыдущему коммиту [git checkout -- .]

Вместо точки указываем нужный файл (index.html)

Удаляем последний созданный коммит [git reset --hard HEAD^1]

Управление ветками

Посмтотреть текущую ветку [git branch]

Создаем новую ветку [git branch develop]

Добавляем название к 'git branch'

Переходим на новую ветку [git checkout develop]

Новую ветку обязательно публикуем, иначе не будет ее на GitHub
